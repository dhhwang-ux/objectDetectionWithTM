# 🤖 AI 휴머노이드 로봇 사물인식 제어 시스템

본 프로젝트는 **Google Teachable Machine**의 이미지 인식 모델과 브라우저의 **Web Serial API**를 결합하여, 인공지능이 사물을 인식하면 실시간으로 휴머노이드 로봇에게 동작 명령(인사 등)을 내리는 웹 기반 제어 시스템입니다.

---

## 🌟 주요 기능

- **실시간 사물 인식**: 웹캠을 통해 사물을 분류하고 확률을 계산합니다.
- **웹 브라우저 직접 제어**: 별도의 설치 프로그램 없이 Chrome/Edge 브라우저에서 로봇과 시리얼(USB) 통신을 수행합니다.
- **지능형 동작 트리거**: 
  - 특정 사물 인식 시 **19번(인사)** 모션 실행.
  - **중복 인사 방지**: 동일한 사물이 계속 감지될 경우 중복 명령을 내리지 않습니다.
  - **동작 보호 시간**: 로봇의 물리적 가동 시간을 고려하여 명령 후 **7초간 대기**합니다.
- **AI 교육 콘텐츠**: 웹페이지 내에 AI와 로봇의 원리를 설명하는 섹션을 포함합니다.

---

## 🛠 시스템 구조

본 시스템은 사용자의 판단(AI)이 하드웨어(Robot)로 전달되는 일련의 과정을 시각화합니다.



1. **Input**: 웹캠을 통해 실시간 영상 입력.
2. **AI Inference**: TensorFlow.js가 브라우저에서 이미지를 분석 및 분류.
3. **Logic**: 확률이 90% 이상이고, 이전과 다른 사물일 경우 명령 생성.
4. **Output**: Web Serial API를 통해 15바이트의 바이너리 명령 패킷을 로봇에 전송.

---

## 📋 설치 및 실행 방법

### 1. 사전 준비
- **브라우저**: 반드시 **Google Chrome** 또는 **Microsoft Edge**를 사용하세요. (Web Serial 지원 필요)
- **하드웨어**: 휴머노이드 로봇과 PC를 USB 케이블로 연결하세요.

### 2. 코드 실행 (중요)
브라우저 보안 정책상 `file://` 경로에서는 시리얼 통신이 차단됩니다. 반드시 **로컬 서버**환경에서 실행해야 합니다.
1. VS Code에서 본 프로젝트 폴더를 엽니다.
2. **Live Server** 확장 프로그램을 설치합니다.
3. `index.html` 우클릭 후 **"Open with Live Server"**를 선택합니다.
4. 주소창이 `http://127.0.0.1:5500/` 형태로 열리는지 확인합니다.

### 3. 사용 단계
1. **로봇 시리얼 연결** 버튼 클릭 -> 로봇이 연결된 포트(COMx)를 선택합니다.
2. **AI 카메라 시작** 버튼 클릭 -> 카메라 권한을 허용합니다.
3. 학습된 사물을 카메라에 보여주면 로봇이 인사를 수행합니다.

## ⚙️ 기술 사양 (Protocol)

로봇에게 전달되는 데이터 패킷은 다음과 같은 15바이트 구조를 가집니다.

| Index | Value | Description |
| :--- | :--- | :--- |
| 0~3 | `0xFF, 0xFF, 0x4C, 0x53` | Header (시작 신호) |
| 11 | `index_char` | **동작 번호 (예: 19 - 인사)** |
| 13 | `100` | 속도/파라미터 |
| 14 | `Checksum` | 데이터 검증 (Index 6~13의 합) |

---

## ⚠️ 주의 사항

- **통신 속도(Baud Rate)**: 로봇이 반응하지 않을 경우 코드 내 `baudRate`를 `115200` 또는 `9600`으로 변경해 보세요.
- **HTTPS/Localhost**: 웹에 배포할 경우 반드시 SSL 인증서(HTTPS)가 적용된 도메인에서만 작동합니다.
- **조명 환경**: Teachable Machine 모델은 조명에 민감하므로 학습 당시와 비슷한 환경에서 테스트하는 것이 좋습니다.
